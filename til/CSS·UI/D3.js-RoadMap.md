### **1일차: Selection API 체감하기 (오늘 할 일)**


    **30분만 투자해서 브라우저 콘솔에서 바로 실험:**


    ```javascript
    // 브라우저 콘솔을 열고 (F12)// 1. 기본 selection 체험
    
    d3.select("body").append("p").text("Hello D3!");
    
    
    // 2. 여러 요소 만들기
    
    const data = [10, 20, 30, 40, 50];
    d3.select("body")
      .selectAll("div")
      .data(data)
      .enter()
      .append("div")
      .style("width", d => d + "px")
      .style("height", "20px")
      .style("background", "blue")
      .style("margin", "2px")
      .text(d => d);
    ```


    **목표:** "아, D3가 DOM을 이렇게 조작하는구나!" 체감하기


### **2일차: 실제 SVG로 바 차트 만들기**


    **CodePen이나 간단한 HTML 파일에서:**


    ```html
    <!DOCTYPE html>
    <html>
    <head>
        <script src="https://d3js.org/d3.v7.min.js"></script>
    </head>
    <body>
        <svg width="500" height="300"></svg>
        <script>
            const data = [30, 80, 45, 60, 20, 90];
            
            d3.select("svg")
              .selectAll("rect")
              .data(data)
              .enter()
              .append("rect")
              .attr("x", (d, i) => i * 70)
              .attr("y", d => 300 - d * 3)
              .attr("width", 60)
              .attr("height", d => d * 3)
              .attr("fill", "steelblue");
        </script>
    </body>
    </html>
    ```


    **목표:** 첫 번째 실제 차트 완성의 성취감 느끼기


### **3일차: Scale 개념 이해**


    **어제 만든 바 차트를 Scale로 개선:**


    ```javascript
    const data = [30, 80, 45, 60, 20, 90];
    
    
    // 스케일 추가 - 이게 D3의 핵심!
    
    const xScale = d3.scaleBand()
        .domain(d3.range(data.length))
        .range([0, 500])
        .padding(0.1);
    
    const yScale = d3.scaleLinear()
        .domain([0, d3.max(data)])
        .range([300, 0]);
    
    d3.select("svg")
      .selectAll("rect")
      .data(data)
      .enter()
      .append("rect")
      .attr("x", (d, i) => xScale(i))
      .attr("y", d => yScale(d))
      .attr("width", xScale.bandwidth())
      .attr("height", d => 300 - yScale(d))
      .attr("fill", "steelblue");
    ```


    **목표:** "Scale이 왜 필요한지" 이해하기

<summary>공부</summary>

⏺ Scale을 넣는 이유:


현재 createDirectlyBarChart에서는 데이터를 직접 픽셀 값으로 사용하고
있습니다:


.attr("height", (d) => d * 4)  // 데이터 30 → 120px 높이


하지만 Scale을 사용하면:

1. 데이터 범위와 화면 크기를 분리

// 스케일 없이 (현재)
height: 30 * 4 = 120px  // 고정 배율


// 스케일로
const yScale = d3.scaleLinear()
.domain([0, 100])        // 데이터 범위 0~100
.range([0, actualHeight]) // 화면 범위 0~400px


height: yScale(30) = 120px  // 비례 계산

1. 자동 비례 계산

// 데이터가 [10, 50, 100]이라면
yScale(10)  = 40px   // 10%
yScale(50)  = 200px  // 50%
yScale(100) = 400px  // 100%

1. 화면 크기 변화에 대응

// 모바일에서 높이 200px일 때
yScale.range([0, 200])
yScale(50) = 100px  // 자동으로 절반 크기


// 데스크톱에서 높이 600px일 때


yScale.range([0, 600])
yScale(50) = 300px  // 자동으로 절반 크기


Scale 없으면: 데이터 값이 바뀔 때마다 수동으로 계산
Scale 있으면: 데이터 범위만 설정하면 자동으로 화면에 맞춰 계산


반응형 + 데이터 독립적 처리가 핵심입니다!
