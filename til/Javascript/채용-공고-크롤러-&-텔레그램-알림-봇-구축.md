## 크롤링이란?


**웹 크롤링(Web Crawling/Scraping)**은 프로그램이 사람 대신 웹사이트에 접속해서 데이터를 자동으로 수집하는 기술이다.


브라우저에서 채용 사이트를 열고 공고를 하나씩 읽는 행위를 코드가 대신 해주는 것.


---


## 프로젝트 전체 구조


```javascript
[원티드/사람인/잡코리아]  →  크롤러(index.js)  →  필터링  →  텔레그램 알림
       데이터 수집            Node.js 서버        조건 매칭      사용자에게 전송
```


---


## 개발 과정


### 1단계: 프로젝트 초기화

- `npm init`으로 Node.js 프로젝트 생성
- **axios**: 웹사이트에 HTTP 요청을 보내는 라이브러리 (브라우저 없이 페이지 데이터를 가져옴)
- **cheerio**: 가져온 HTML을 파싱하는 라이브러리 (jQuery처럼 CSS 셀렉터로 원하는 데이터를 추출)

### 2단계: 3개 플랫폼별 크롤링 함수 작성


| 플랫폼  | 방식             | 설명                                                            |
| ---- | -------------- | ------------------------------------------------------------- |
| 원티드  | API 호출         | 공개 API에서 JSON 데이터를 바로 받아옴. HTML 파싱 불필요                        |
| 사람인  | HTML 크롤링       | axios로 HTML 요청 → cheerio로 CSS 셀렉터 파싱. User-Agent 헤더로 차단 방지    |
| 잡코리아 | Next.js SSR 파싱 | Next.js 구조라 일반 크롤링이 안 됨. self.__next_f.push 청크와 a 태그에서 데이터 추출 |

- 검색 키워드: `프론트엔드`, `React`, `Next.js` (3개 키워드로 확장하여 수집량 증가)

### 3단계: 필터링 로직


수집한 공고를 두 가지 트랙으로 필터링:


**목표 트랙 (대기업/금융권 키워드)**

- 삼성, LG, 현대, SK, 카카오, 네이버, 토스 등
- KB, NH, 신한, 하나, 우리 등 금융사
- 쿠팡, 배민, 야놀자, 무신사 등 IT 플랫폼

**현실 트랙 (광주/전남 지역)**

- 광주, 전남, 나주, 목포, 순천, 여수 지역 공고는 회사 무관하게 통과

**공통 조건: 경력 3년차 지원 가능 여부**

- "경력 무관", "신입/경력", "3년 이상" 등 텍스트를 정규식으로 판별

### 4단계: 중복 방지

- `sent_jobs.json`에 `wanted_12345`, `saramin_6789` 형태로 전송 이력 저장
- 다음 실행 시 이미 보낸 공고는 건너뜀

### 5단계: 텔레그램 알림 전송

- Telegram Bot API의 `sendMessage` 호출
- HTML 포맷으로 회사명, 공고 제목, 경력, 지역, 링크 전송
- 각 메시지 사이 1초 딜레이 (API 속도 제한 방지)

### 6단계: GitHub Actions 자동화

- `.github/workflows/crawler.yml` 파일로 설정
- **cron:** **`0 */2 * * *`** → 2시간마다 GitHub 서버에서 자동 실행
- 컴퓨터를 꺼도 클라우드에서 동작
- `sent_jobs.json` 변경 시 자동 커밋/푸시

## 트러블슈팅


| 문제                        | 원인                                   | 해결                                   |
| ------------------------- | ------------------------------------ | ------------------------------------ |
| 잡코리아 0개 수집                | Next.js SSR 구조로 일반 HTML 셀렉터가 작동하지 않음 | GI_Read 링크 + 주변 텍스트에서 데이터 추출 방식으로 변경 |
| GitHub Actions Node 18 에러 | undici 패키지가 Node 20 이상 필요            | Node 22로 업그레이드                       |
| git push 403 에러           | 워크플로우에 쓰기 권한 없음                      | permissions: contents: write 추가      |
