> 📅 **학습일**: 2026-06-08

> 💡 **프로젝트S v3** — Server·Network·Storage 통합 인프라 모니터링·관리 시스템의 프론트엔드. NOC 운영자가 마우스 없이도 빠르게 인프라 상태를 순회·진단할 수 있도록 **접근성·반응형·실시간 시각화**에 집중했다.

## 📌 한 줄 요약


대규모 인프라 모니터링 SaaS의 프론트엔드를 **웹 접근성(WCAG)·UI 스케일링·대시보드 위젯 시스템·실시간 데이터 시각화** 4축으로 고도화하고, 인증·배포 단계의 사용자 이탈을 유발하던 안정성 버그를 근본 원인까지 파고들어 해결했다.


---


## 🏆 대표 성과 (3줄 요약)

1. **전 모니터링 화면 키보드 접근성 일괄 확보** — WAI-ARIA roving tabindex 공용 훅(`useListKeyboardNav`·`useFocusTrap`)을 **27개 컴포넌트**에 적용해 WCAG 2.1.1(Keyboard) 충족, NOC 운영자의 마우스 없는 조작 지원.
2. **저시력 대응 UI 스케일링** — 글자 크기 **5단계(0.875~1.25배)** 전체 스케일링을 위해 px 하드코딩을 **71개 파일**에서 rem/em으로 전환, 루트 CSS 변수 단일 제어.
3. **인증·배포 안정성 근본 해결** — 동시 401 자동로그인 끊김(singleflight 직렬화), 배포 중 흰 화면(chunk error recovery + 404)을 증상 때우기가 아닌 근본 원인까지 추적해 수정.

## 🎯 핵심 성과 (STAR)


### ① 전 화면 웹 접근성 — 키보드 네비게이션 체계화

- **Situation** — NOC 운영자는 장시간 모니터링 중 마우스보다 키보드로 빠르게 행을 순회·진단해야 하지만, 테이블·리스트·카드·토폴로지가 모두 `onClick` 마우스 전용이었다.
- **Task** — 화면마다 제각각 구현하지 않고 **재사용 가능한 접근성 패턴**으로 전 화면에 일관 적용한다.
- **Action**
    - WAI-ARIA **roving tabindex** 패턴의 `useListKeyboardNav` 훅 구현 — ArrowUp/Down 순환, Home/End 점프, Enter/Space 활성화, Tab 한 번에 리스트 탈출
    - 카드뷰용 **2차원 그리드 키보드 네비**로 확장, 토폴로지·랙·수집 설정까지 적용
    - `useFocusTrap` + **모달 닫힘 후 직전 항목으로 포커스 복귀(focusActive)** 로 포커스 컨텍스트 유지
    - 폼 검증 실패 시 **첫 invalid 필드로 자동 포커스 이동**
- **Result** — WCAG 2.1.1(Keyboard) 충족. 장애/이벤트/사용자/상세 모달/변경 이력 등 전 모니터링 화면이 마우스 없이 조작 가능해짐. 마운트 시 자동 포커스를 의도적으로 배제해 다른 focus trap과의 충돌도 사전 차단.

### ② 저시력 대응 UI 스케일링 — 글자 크기 5단계 전체 조절


*(이미지 생략)*


*(이미지 생략)*

- **Situation** — 관제실 대형 모니터·다양한 시력 환경에서 고정 px 기반 UI는 가독성 편차가 컸다.
- **Task** — 사용자가 직접 전체 UI 배율을 조절하고, 그 설정이 모든 컴포넌트·아이콘·차트에 비례 반영되게 한다.
- **Action**
    - `xs~xl` **5단계 배율(0.875 ~ 1.25)** 정의 → 루트에 CSS 변수 `--font-scale` 와 `data-font-scale` 속성으로 노출(`useLayoutEffect`로 페인트 전 적용, 깜빡임 없음)
    - 설정은 Zustand `persist`로 영속화 → 재접속 시 유지
    - **px 고정값 → rem/em 대규모 마이그레이션** (Header·Sidebar·아이콘·상세 패널·랙 다이어그램·트렌드 타임라인·대시보드 그리드 등 전 영역)
    - `xl` 단계에서 위젯 값/라벨을 가로 배치로 분기하는 등 **단계별 레이아웃 대응**
- **Result** — 한 번의 토글로 전 UI가 비례 스케일링. 텍스트 넘침·레이아웃 깨짐 회귀를 단계별로 잡아내며 디자인 토큰 일관성도 함께 정비.

### ③ 커스텀 대시보드 위젯 시스템

*(이미지 생략)*

- **Situation** — 사용자마다 보고 싶은 인프라 지표가 달라, 직접 위젯을 배치·편집하는 대시보드가 필요했다.
- **Task** — 드래그 배치 + 반응형 + 인터랙션 밀도가 높은 편집 경험을 구현한다.
- **Action**
    - `react-grid-layout` 기반 위젯 편집(드래그 리사이즈·배치), **글자 스케일 연동 그리드 props**
    - 메뉴 히스토리 탭 재정렬을 `dnd-kit/sortable`로 전환해 **모바일 터치 드래그** 지원

    - 숫자 **카운트업·진입 애니메이션**, 그리드 셀에 맞춘 **skeleton 높이**로 로딩 점프 제거
- **Result** — 편집·조회 모두 매끄러운 위젯 대시보드 완성. 드래그 새로고침 시 화살표 소실, 편집 페이지 추종 모드 미반영 등 엣지 버그까지 정리.

- **Situation** — 네트워크 토폴로지·장비 상태·트렌드를 직관적으로 보여줘야 했다.
- **Action**

    - **트렌드 차트** — 비교 식별자 멀티셀렉트 공용화(키보드 네비 포함), SAN 스위치 위젯군 추가
- **Result** — 초기 렌더 깜빡임·줌 처리 등 미세 결함까지 다듬은 운영용 시각화 화면 제공.

### ⑤ 인증·배포 안정성 (트러블슈팅)

- 배포 중 청크 교체로 인한 **흰 화면** → `lazyWithRetry` + 404 페이지로 자동 복구 (아래 상세)

### ⑦ 네트워크 스위치 모니터링 페이지 신규 구축

- **Situation** — 서버·SAN에 이어 네트워크 스위치 영역에는 통합된 모니터링 화면이 없어, 운영자가 스위치 상태·포트·트렌드를 한 화면에서 진단할 수 없었다.
- **Task** — 전체현황·장치목록·상세팝업을 갖춘 모니터링 페이지를 형제 페이지(서버·SAN) 패턴과 공용 컴포넌트를 재사용해 구축한다.
- **Action**
    - 전체현황 KPI 타일 + 하단 최근 이벤트 섹션, 주의 장치 카드(0대 시 `EmptyState`), 임계값 색상 기반 CPU·메모리·에러 메트릭 카드/테이블
    - 상세 팝업 **4탭(정보·물리·트렌드·이벤트)** — 포트를 **LAG/Pair 상태 LED 그리드·페이스플레이트**로 시각화
    - 사용률 인라인 바를 `UsageBar` 공용 컴포넌트로 추출, BGP 피어·Down/에러 인터페이스 **건강도 추이 위젯 3종** 추가
- **Result** — 네트워크 스위치를 서버·SAN과 동일한 깊이로 진단 가능한 화면 완성. 섹션별 검색바·툴팁·모바일 가로 스크롤까지 정돈.

### ⑧ 네트워크 토폴로지 그래프 고도화

- **Situation** — 토폴로지가 단일 레이아웃·평면 표시라 SDN/VTEP 계층 구조와 대규모 노드를 직관적으로 보여주지 못했다.
- **Action**
    - **트리/존 2모드 레이아웃 + 상태 시각화 개편**, SDN 클러스터 기준 컨트롤러 **자동 집계**, VTEP **노드↔피어 양방향 선택 동기화**
    - 시스템 구성도·네트워크 토폴로지·대시보드 위젯 **전체 화면 보기**, 노드 클릭 시 장치 상세 팝업 연동
    - 패브릭 배경 가시성 개선, info 패널·툴팁 props 분리, 초기 진입 중앙 fit
- **Result** — 미연결 패널 정리 등 엣지까지 다듬은 계층형 토폴로지 제공.

### ⑨ 대시보드 위젯 시스템 확장

    - 장비 **허니콤 위젯 축소 시 단일 통합 compact 모드**, SDN 인터페이스 **혼잡 히트맵 위젯**(차트 색상 status 기반 통일)

- **Result** — 위젯 종류·밀도 확장과 리사이징 자유도 동시 확보.

### ⑩ 멀티 테마 status 색상 가시성 정합

- **Situation** — 라이트/다크/고대비 등 테마별로 status(정상·주의·위험) 색상 대비가 들쭉날쭉해 고대비 환경 가독성이 떨어졌다.
- **Action** — **테마 5개 파일**의 status 색상·배경 대비 일괄 재정의, 고대비 테마 상태 뱃지/글씨 두께 보정, 토폴로지·SAN 광케이블 위젯 이미지 밝기 보정.
- **Result** — 전 테마에서 상태 구분이 명확해지고 고대비 접근성 충족.

### ⑪ 모니터링 전 화면 모바일/태블릿 반응형

- **Action** — 네트워크 스위치 상세 팝업 트렌드 툴바·검색바·stat 칩(2행 가로 스크롤)·도넛+표 섹션 넘침, 탭바 슬롯 정렬, 수집 설정 반응형 등 모바일 깨짐을 영역별로 해소.
- **Result** — 데스크톱 전용이던 모니터링 화면이 태블릿/모바일에서도 조작 가능해짐.

### ⑫ 메뉴 라우트 히스토리 — 브라우저 탭형 네비게이션

- **Situation** — NOC 운영자는 장애·이벤트·장비 상세를 오가며 같은 메뉴를 반복 순회하는데, SPA에는 브라우저 탭 같은 "방문한 화면 되돌아가기" 수단이 없었다.
- **Task** — 방문 메뉴를 탭으로 쌓아 재방문·재정렬·일괄 정리가 가능한 히스토리 바를, 키보드·터치 접근성까지 갖춰 구현한다.
- **Action**
    - 방문 메뉴를 **LRU 스택(최대 15개)** 으로 쌓는 `menuRouteHistoryStore`, 라우트 변경을 `MainLayout` 이 감지해 동기화
    - **ResizeObserver 기반 오버플로우 분리** — 컨테이너 폭 구간으로 visible 개수를 정하고 초과분은 `+N` OverflowMenu 로. 활성 탭이 hidden 으로 밀리면 마지막 visible 탭과 swap 해 **항상 보이도록** 보장
    - **WAI-ARIA Tabs 패턴(Manual activation)** — roving tabindex, Arrow/Home/End 는 포커스만 이동하고 navigate 는 Enter/Space·클릭 시점에만 (SPA 라우트 전환 비용이 커 자동 활성화 배제, W3C APG 권장)
    - **dnd-kit 마우스+터치 재정렬** — MouseSensor(distance 8px)로 클릭과 드래그 구분, TouchSensor(delay 200ms)로 스크롤과 충돌 방지. KeyboardSensor 는 tablist 키보드 동작과 충돌하므로 의도적으로 제외하고 `attributes` 미부착
    - 활성 탭 닫기 시 **인접 LRU 탭으로 자동 이동** 후 제거, 전체 비우기는 제거 대상 개수를 노출한 **확인 다이얼로그**로 오조작 방지
    - font-scale 연동(px→rem/em)으로 글자 배율에 비례
- **Result** — 마우스·키보드·터치 어디서든 조작 가능한 탭형 네비게이션 완성. 접근성(roving tabindex)과 라이브러리 기본 동작(dnd-kit attributes)의 충돌을 구조적으로 분리한 점이 핵심.

### ⑬ 보안 위협 모니터링 (보안벤더P 보안 장비)

- **Situation** — 보안 장비(보안벤더P)의 공격 탐지가 단순 카운트로만 노출돼, 운영자가 위협의 심각도와 추세를 한눈에 파악하기 어려웠다.
- **Action**
    - 보안 공격 유형을 **심각도 등급으로 분류**하고 탐지 분포를 차트로 시각화
    - 장비 상세 정보 탭에 **보안 공격 유형·로그 섹션** 신설
    - 최근 보안 탐지를 위협 과장 없이 **정보성 알림 배너**로 전환해 오탐 피로도 완화
- **Result** — 네트워크·서버 중심이던 모니터링에 **보안 관점**을 추가, 공격 유형·심각도·로그를 한 화면에서 진단 가능.

### ⑭ 네트워크 스위치 운영 리포트

- **Situation** — 실시간 화면만으로는 일정 기간의 트래픽·에러 추이를 회고적으로 분석하기 어려웠다.
- **Action** — 트래픽·에러를 **하나의 통합 차트**로 겹쳐 보여주고, 하단에 **일시별 상세 표**를 붙여 수치 단위로 드릴다운하도록 구성(기간 선택 연동).
- **Result** — 운영자가 스위치의 기간별 트래픽/에러 패턴을 리포트 형태로 점검 가능.

### ⑮ 컴포넌트 단위 렌더 에러 격리 강화

- **Situation** — 차트·모달 내부 한 위젯의 렌더 에러가 화면 전체(또는 모달 전체)를 흰 화면으로 무너뜨렸다.
- **Action** — **차트 단위 ErrorBoundary**로 개별 차트 실패를 그 자리에서만 fallback 처리하고, 모달은 children 렌더 에러를 **body 안쪽으로 격리**해 헤더·닫기 버튼은 살아있게 분리.
- **Result** — 부분 실패가 전체 장애로 번지지 않는 구조 확보. 트래픽 차트 로딩 시 EmptyState 깜빡임·에러 지연 렌더 등 부수 결함도 함께 제거.

### ⑯ 공용 날짜·시간 입력기 확장 & 벤더 스키마 robust 처리

- **Action**
    - 기존 DatePicker에 **TimePicker·DateTimePicker·DateTimeRangePicker**를 추가하고 DatePicker **선택 범위 제한**을 지원 — 등록/수정·검색 조건 전반의 입력기를 일원화
    - 네트워크 스위치 상태 코드(숫자/문자열 혼재)·인터페이스 IP·센서 상태를 **robust 하게 정규화**하고, 벤더별 필드명 차이(`ifIpAddress`·`cpuUsage`·`mfgName` 등)를 **폴백 체인으로 흡수**
- **Result** — 입력 UX 일관성 확보 및 벤더·수집기마다 다른 데이터 표현을 표준 모델로 수렴.

### ⑰ 네트워크 토폴로지 force-directed 재설계

- tier 자동분류 체계를 폐기하고 **force-directed 연결도 레이아웃**으로 전환. 허브 노드 펄스·글로우 모션과 WARNING 호흡 링 등 과한 효과를 정리해 대규모 노드에서도 연결 관계가 직관적으로 드러나도록 개선.

## 🔧 트러블슈팅 하이라이트


### 🔁 동시 401 → 자동 로그인 끊김 (singleflight)


**증상** — 토큰 만료 시점에 여러 API가 동시에 401을 받으면 로그인 페이지로 튕김.


**근본 원인** — 백엔드가 refresh 토큰을 **회전(rotation)** 시키는데, 동시 401이 각각 refresh를 호출 → 두 번째 요청이 이미 폐기된 옛 토큰으로 요청 → 401 + 방금 갱신된 새 토큰까지 삭제.


**해결** — 모듈 레벨 `refreshPromise` 하나로 refresh를 **단 한 번만 실행**하고, 동시 401들은 같은 Promise를 await해 결과를 재사용. 성공/실패와 무관하게 완료 후 `refreshPromise = null`로 해제해 다음 만료 주기를 보장.


```typescript
let refreshPromise: Promise<string> | null = null;

function refreshAccessToken(refreshToken: string): Promise<string> {
  if (!refreshPromise) {
    refreshPromise = axios.post(`${API_BASE_URL}/api/auth/refresh`, { refreshToken })
      .then((res) => { /* 새 토큰 저장 */ })
      .finally(() => { refreshPromise = null; });
  }
  return refreshPromise; // 동시 401은 같은 Promise를 공유
}
```


### ⚪ 배포 중 흰 화면 (chunk error recovery)


**증상** — 새 버전 배포 직후, 이전 번들을 들고 있던 사용자가 동적 import 청크 로드에 실패해 빈 화면.


**해결** — `lazyWithRetry`로 청크 로드 실패를 감지해 **1회 자동 새로고침**(최신 번들 fetch). `sessionStorage` 플래그로 **무한 리로드 루프를 방지**하고, 리로드 후에도 실패하면 ErrorBoundary로 전파 → 404/에러 페이지로 graceful degradation.


### 🔤 px 고정 → rem/em 마이그레이션 (Font Scale)


사용자 배율을 전 컴포넌트에 비례 반영하기 위해 px 하드코딩을 rem/em으로 전환. 루트 CSS 변수 하나로 전체를 제어하되, 단계별로 레이아웃이 깨지지 않도록 `data-font-scale` 속성 분기를 병행. **"한 곳만 바꾸면 전부 따라오는" 구조**로 설계해 유지보수 비용을 낮춤.

## 🧠 회고 & 배운 점

- **접근성은 기능이 아니라 설계** — 화면별로 덧붙이는 대신 훅으로 패턴화하니 일관성·재사용성·회귀 방지가 동시에 따라왔다.
- **버그는 증상이 아니라 근본 원인** — 401 튕김을 "재시도"로 덮지 않고 토큰 회전이라는 진짜 원인을 찾아 singleflight로 구조적으로 해결.
- **전역 변수 하나의 힘** — font-scale CSS 변수처럼 단일 제어점을 두면 광범위한 변경도 유지보수 가능해진다.
- **다중 인스턴스·실시간 환경 전제** — 로컬 상태 의존을 피하고 사용자 설정·세션을 외부에 영속화하는 습관.
