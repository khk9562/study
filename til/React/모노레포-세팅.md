## 모노레포 react+vite+turbo 생성


    모노레포를 Turborepo를 사용하여 React와 Vite로 세팅된 프로젝트 두 개를 만드는 방법을 설명해 드리겠습니다. 단계별로 진행하겠습니다:

    1. 새 디렉토리 생성 및 초기화:

    ```shell
    mkdir my-monorepo
    cd my-monorepo
    npm init -y
    ```

    1. Turborepo 설치:

    ```shell
    npm install turbo -D
    ```

    1. 루트 `package.json` 파일 수정:

    ```json
    {
      "name": "my-monorepo",
      "version": "1.0.0",
      "private": true,
      "workspaces": ["apps/*"],
      "scripts": {
        "dev": "turbo run dev",
        "build": "turbo run build"
      },
      "devDependencies": {
        "turbo": "^1.x.x"
      }
    }
    ```

    1. `turbo.json` 파일 생성:

    ```json
    {
      "$schema": "<https://turbo.build/schema.json>",
      "pipeline": {
        "build": {
          "outputs": ["dist/**"]
        },
        "dev": {
          "cache": false
        }
      }
    }
    ```

    1. 두 개의 Vite + React 프로젝트 생성:

    ```shell
    mkdir -p apps/project1
    mkdir -p apps/project2
    cd apps/project1
    npm init vite@latest . -- --template react
    cd ../project2
    npm init vite@latest . -- --template react
    cd ../..
    ```

    1. 각 프로젝트의 `package.json` 파일에서 `"private": true`를 추가하고 이름을 변경합니다:

    project1/package.json:


    ```json
    {
      "name": "project1",
      "private": true
      // ...나머지 내용
    }
    ```


    project2/package.json:


    ```json
    {
      "name": "project2",
      "private": true
      // ...나머지 내용
    }
    ```

    1. 루트 디렉토리에서 의존성 설치:

    ```shell
    npm install
    ```


    이제 모노레포가 설정되었습니다. 다음 명령어로 프로젝트를 실행할 수 있습니다:

    - 개발 모드로 모든 프로젝트 실행: `npm run dev`
    - 모든 프로젝트 빌드: `npm run build`

    각 프로젝트는 독립적으로 실행할 수도 있습니다:


    ```shell
    npm run dev --workspace=project1
    npm run dev --workspace=project2
    ```


## 공통 패키지 관리 - bootstrap 및 redux 설치

    1. 먼저 루트 디렉토리에 `packages` 폴더를 만들고, 그 안에 공통 패키지를 위한 폴더를 생성합니다:

    ```shell
    mkdir -p packages/shared
    cd packages/shared
    npm init -y
    cd ../..
    ```

    1. `packages/`**`shared`**`/package.json` 파일을 다음과 같이 수정합니다:

    ```json
    {
      "name": "@my-monorepo/shared",
      "version": "1.0.0",
      "private": true,
      "main": "index.js",
      "dependencies": {
        "@reduxjs/toolkit": "^2.0.1",
        "react-redux": "^9.1.0",
        "bootstrap": "^5.3.2"
      }
    }
    ```

    1. **루트** **`package.json`** **파일**의 `workspaces` 필드를 수정하여 `packages` 폴더를 포함시킵니다:

    ```json
    {
      "name": "my-monorepo",
      "version": "1.0.0",
      "private": true,
    
      "workspaces": [
        "apps/*",
        "packages/*"
      ],
    
      "dependencies": {
        "@reduxjs/toolkit": "^2.0.1",
        "react-redux": "^9.1.0",
        "bootstrap": "^5.3.2"
      }
    }
    ```

    1. 루트 디렉토리에서 의존성을 설치합니다:

    ```shell
    npm install
    ```

    1. 이제 각 프로젝트(`apps/project1`과 `apps/project2`)의 `package.json` 파일에 공유 패키지를 의존성으로 추가합니다:

    ```json
    {
      "dependencies": {
        "@
    my-monorepo
    /shared": "^1.0.0"
        // 다른 의존성들...
      }
    }
    ```

    1. 루트 디렉토리에서 다시 `npm install`을 실행하여 프로젝트 간 링크를 설정합니다.

    → 이 과정에서 에러가 많이 났다.
    package.json은 루트/shared/각 프로젝트 안에 있는 총 4개이며,
    반드시 레포지토리 이름을 오타없이 통일시켜주어야한다. (@**my-monorepo**)
    이후, 모든 `node_modules` 폴더와 루트의 `package-lock.json` 파일을 삭제하고, 루트 디렉토리에서 다시 `npm install` 실행
    

    1. 이제 각 프로젝트의 코드에서 공유 패키지의 의존성을 직접 import할 수 있습니다. 예를 들어:

    ```javascript
    import { configureStore } from '@reduxjs/toolkit';
    import { Provider } from 'react-redux';
    import 'bootstrap/dist/css/bootstrap.min.css';
    ```


## SCSS 및 react-bootstrap 공용 세팅 및 사용

    1. vite.config.js에서 additionalDAta 부분 주석처리

    ```javascript
    import { defineConfig } from "vite";
    import react from "@vitejs/plugin-react";
    import path from "path";
    
    // https://vitejs.dev/config/
    export default defineConfig({
      plugins: [react()],
      css: {
        preprocessorOptions: {
          scss: {
            // additionalData: `@import "@monorepo-test-turbo/shared/styles/main.scss";`,
          },
        },
      },
      resolve: {
        alias: {
          "@": path.resolve(__dirname, "./src"),
          "@monorepo-test-turbo/shared": path.resolve(
            __dirname,
            "../../packages/shared"
          ),
        },
      },
    });
    ```

    1. shared/main.scss에서

        ```javascript
        @import "bootstrap/scss/bootstrap";
        ```

    2. 각 프로젝트의 main.jsx에서

        ```javascript
        import "@monorepo-test-turbo/shared/styles/main.scss";
        ```
