### 요구사항

- 현장에서 예약 후에 프린트하는 로직이 필요한데, 프린트 요청 보내는 코드가 로컬 파이썬 서버. 그래서 관리자 예약(==현장 예약결제) 후에 프린트 요청을 보낼 때 웹 소켓을 열어 요청을 보내야함.

### Web Socket이란?

- 웹 앱과 서버 간의 지속적인 연결을 제공하는 프로토콜.
- 서버와 클라이언트 간에 양방향 통신이 가능.
- HTTP와 달리 WebSocket은 한 번 열린 후 계속 유지되므로, 서버나 클라이언트에서 언제든지 데이터 전송 가능

### Web Socket의 특징

- `실시간 통신`
- `효율성` : 지속적인 연결을 통해 데이터를 교환할 때 발생하는 오버헤드(프로그램의 실행 흐름에서 나타나는 현상)가 줄어듭니다.
- `서버에서 클라이언트로 직접 푸시 가능`
클라이언트가 지속적으로 서버에 폴링하지 않아도 되므로, 리소스 사용 최소화하고 반응성을 향상시킴
- `낮은 지연 시간` : ws 연결은 초기 핸드쉐이크(통신에서 연결을 설정하기 위한 과정)가 완료된 후에는 지속적으로 열려 있으므로, 데이터 전송에 필요한 지연 시간이 크게 줄어듦
- `양방향 통신` : HTTP의 단방향 요청-응답 모델과 대조적으로, 클라이언트와 서버 모두에서 데이터를 송신하고 수신할 수 있음
- `유연성` : ws는 포트 80 및 443을 통해 통신하므로, 대부분의 방화벽 및 프록시 서버에서 차단되지 않습니다.
