1. 비동기

```javascript
const sendPrintRequest = (trigger, content) => {
    if (!socketRef.current || !isConnected) {
      console.error("📡 웹소켓 연결이 활성화되지 않았습니다")
      return Promise.reject(new Error("📡 웹소켓 연결이 활성화되지 않았습니다"))
    }

    return new Promise((resolve, reject) => {
      try {
        const message = JSON.stringify({
          trigger,
          content,
        })

        socketRef.current.send(message)
        resolve(true)
      } catch (error) {
        console.error("📡 메시지 전송 오류:", error)
        reject(error)
      }
    })
  }
```


Promise는 비동기 작업을 위한 것으로, 나중에 완료될 작업을 처리할 수 있습니다.


### 비동기(Asynchronous)

- 작업이 완료되기를 기다리지 않고 다음 작업을 실행합니다.
- 실생활 예시: 식당에서 주문 후 진동벨을 받고 다른 일을 하다가 벨이 울리면 음식을 가져가는 것

```javascript
// 비동기 방식 예시
console.log("1. 주문 시작");
prepareCoffee().then(result => {  
// 커피 준비 요청 후 다른 작업 진행
    console.log("3. 커피 완료:", result);
});
console.log("2. 다른 작업 진행");

// 실행 순서: 1 -> 2 -> 3
```


### 비동기가 필요한 이유:

1. 성능: 긴 작업을 기다리는 동안 다른 작업을 처리할 수 있음
2. 사용자 경험: 웹사이트에서 데이터를 불러오는 동안 UI가 멈추지 않음
3. 자원 효율성: 시스템 자원을 더 효율적으로 사용할 수 있음

즉, 작업이 완료되는데 시간이 오래 걸리거나, 언제 완료될지 모르는 경우(예: 네트워크 요청), 또는 주기적으로 수행해야 하는 작업의 경우 비동기 처리가 적합합니다.

1. 동기

```javascript
const sendPrintRequest = (trigger, content) => {
  if (!socketRef.current || !isConnected) {
    throw new Error("📡 웹소켓 연결이 활성화되지 않았습니다")
  }

  try {
    socketRef.current.send(message)
    return true
  } catch (error) {
    throw error
  }
}
```


일반 try-catch는 동기적으로 즉시 실행되고 결과를 반환합니다.


### 동기(Synchronous)

- 작업이 순차적으로 실행되며, 하나의 작업이 완료될 때까지 다음 작업이 기다립니다.
- 실생활 예시: 은행 창구에서 줄을 서서 순서대로 업무를 처리하는 것

```javascript
// 동기 방식 예시
console.log("1. 주문 시작");
const 커피준비 = prepareCoffee();  // 이 작업이 끝날 때까지 기다림
console.log("2. 커피 완료:", 커피준비);
console.log("3. 주문 완료");

// 실행 순서: 1 -> 2 -> 3
```


→ WebSocket의 `send()` 메서드는 비동기적으로 동작할 수 있으므로, Promise를 사용하는 것이 더 적절할 수 있습니다.


현재의 경우, 실제로 `send()` 메서드가 동기적으로 동작하고 별도의 비동기 처리가 필요 없다면 일반 try-catch 방식으로도 충분할 것 같습니다. Promise는 불필요한 복잡성을 추가할 수 있습니다.
