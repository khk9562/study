## 하네스 엔지니어링이란


`하네스(harness) = "코드를 직접 만들진 않지만, AI가 만들도록 둘러싸인 환경 전체"입니다.` 좀 더 풀면:


▎ AI 에이전트가 일관되고 예측 가능한 결과를 내도록, 그 주위에 규칙·문서·도구·워크플로우를 구조화해 놓는 작업


엔진(=AI 모델) 자체를 바꿀 수는 없으니, 엔진을 어떤 `마구(harness)`로 묶어서 어디로 끌고 갈지 설계하는 일이에요. 같은 모델이라도 하네스가 잘 잡힌 레포에서 일하면 결과 품질이 몇 배 차이 납니다.

## 왜 사용하는가


AI는 "기억력 없는 천재 인턴"입니다. 매 세션이 백지에서 시작되고, 코드는 잘 짜지만 이 회사의 룰을 모릅니다.
하네스가 없으면:

- 같은 실수를 매번 반복 (단위, 네이밍, 권한 처리 누락)
- 사람마다 결과 품질 다름 (지시를 잘 한 사람만 좋은 결과)
- 새 사람/새 기능마다 매번 처음부터 설명
- "괜찮아 보이는" 코드가 사실 컨벤션 위반인 채로 머지됨

하네스를 만들면:

- 재현성 — 누가 시키든 같은 품질
- 위임 한계 확장 — 큰 작업도 안전하게 맡길 수 있음
- 온보딩 가속 — 사람도 AI도 같은 문서로 학습 (지금 본인의 상황!)
- 시스템적 학습 — 실수가 발견될 때 문서/규칙 보강 → 다음부터 안 일어남

## 효율적으로 쓰는 법


원칙 1 — "한 번 말한 건 두 번 말하지 않는다"


같은 지시를 두 번 이상 하게 되면 그건 코드가 아니라 문서/규칙으로 박을 신호입니다. 매번 "MiB로 계산해줘"라고 말하는 대신 docs/domain/data-units.md에 적어두는 거죠.


원칙 2 — 규칙은 짧게, 지식은 깊게

- [CLAUDE.md/AGENTS.md](http://claude.md/AGENTS.md) = "절대 어기면 안 되는 것"만. 길어지면 무시당함.
- docs/ = 깊은 내용은 여기. 필요할 때 AI가 찾아 읽음.
- 인덱스([README.md](http://readme.md/)) 패턴으로 "어디에 뭐가 있는지"만 짧게 노출.

원칙 3 — 워크플로우를 명문화


"읽기 → 짜기 → 검증" 순서를 Step으로 박아두면 AI가 단계를 건너뛰지 않습니다. 이 프로젝트의 Step 1~5가


정확히 그것.


원칙 4 — 검증은 자동화 가능한 명령으로


"빌드/린트가 통과해야 끝"이라는 룰이 있고 명령어가 명시돼 있으면 AI가 스스로 검증·수정합니다. 이게 없으면


"다 됐어요"라고 말하고 깨진 코드를 두고 갑니다.


원칙 5 — 발견-반영 루프 만들기


하네스는 한 번에 완성되지 않습니다. AI가 헛다리 짚는 패턴을 발견할 때마다 문서/규칙에 반영하면 시스템이 자가 진화합니다.

### 워크플로우(Routine) — Step 1~5


CLAUDE.md의 5단계가 작업 루틴을 강제합니다:

1. 작업 유형 파악
2. 관련 문서 읽기 ← (영역별 매트릭스 제공)
3. 코드 작성
4. 문서 검토 ← 도메인 지식 변경 시 docs 갱신
5. 빌드 검증 ← 명령어까지 명시

Step 4가 특히 영리함: AI가 만든 변경이 다시 문서로 환류되어 하네스를 강화합니다.

## 탐색 메모 (raw)

- 대시보드 위젯 드래그 후 새로고침하면 위치 초기화. /dashboard/custom
- 모니터링 진입 시 차트가 1번 깜빡 후 사라짐
- 이벤트 목록에서 GiB랑 GB 섞여있음
- 시스템 관리 → 사용자 페이지에서 검색창 폭이 너무 좁음

이 정도면 충분합니다. 흐름 끊기는 게 가장 손해라서 형식 신경쓰지 마세요.


탐색 후 AI에게 한 번에 위임


이 raw 메모를 todo/ui-ux-review.md에 정리해줘.


형식은 [P/카테고리/제목] + 재현/파일/의도/제약/완료기준 템플릿.


관련 파일과 docs 링크는 직접 찾아서 채우고,


의도와 우선순위는 비어있으면 "TBD"로 두고 나에게 질문해줘.


AI가 할 일:

- 각 항목의 관련 파일을 grep으로 찾아 경로 박기
- 관련 docs 링크 추가 (예: 단위 이슈 → docs/domain/data-units.md)
- 다중 인스턴스, 권한 등 제약 후보 자동 추가
- 모르는 부분은 "TBD" + 질문으로 남기기 (이게 중요)

사람은 검수만


AI가 채운 결과 보고:

- 우선순위 확정
- "TBD" 빈칸 채우기
- 의도가 본인 머릿속과 다르면 1줄 수정

안 되는 것


AI한테 **그냥 "todo 만들어줘"**라고만 하면 헛것을 채웁니다. 본인이 직접 보지 않은 이슈를 상상으로


만들어내거나, 우선순위를 임의로 매겨요.


규칙: 사람은 raw signal을 주고, AI는 정형화·확장만.


발견 → 형식화 사이클이 잡히면, 본인 시간의 90%는 탐색과 의도 결정에 쓰고, 10%만 검수에 쓰면 됩니다. MD
작성 자체에 시간 쓰는 건 비효율이에요.


한 단계 더 — 자주 쓰는 지시는 명령어로
