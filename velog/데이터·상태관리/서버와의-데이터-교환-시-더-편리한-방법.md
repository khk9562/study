서버에서 배열 객체 형태로 데이터를 보내주면,
이것을 사용자에게 보여주고
보여준 내용을 토대로 사용자가 수정을 한다면,
그 수정내용을 다시 서버로 데이터를 보내줘야한다.

처음 이 수정과정을 경험했을 때는, 배열 안의 모든 객체의 키값을 구분하여 state를 노가다로 만들어 input의 name을 이용하여 일일이 저장하고, 다시 서버로 보내는 미친 짓을 했었다.

하지만 지금은 그 배열 객체를 하나의 state에 넣고, 아래와 같이 더 보편적인 함수를 이용한다.


```
const handleInputChange = (
    field: string,
    value: string | boolean | number
  ) => {
    setData((prev) => ({ ...prev, [field]: value }));
  };
```



그리고 만약 서버 측에서 boolean과 number 값도 String 처리해서 보내주길 원한다면,
fetch의 body에 아래와 같이 적용해서 보낸다.
```
const settingsbody = Object.entries(pfData).reduce((obj, [key, value]) => {
      return {
        ...obj,
        [key]:
          typeof value === "number" || typeof value === "boolean"
            ? String(value)
            : value,
      };
    }, {});
```

이것도 최선의 방법은 아니라고 확신한다. 더 나은 방법이 생기면 추가 업데이트 하겠다.
